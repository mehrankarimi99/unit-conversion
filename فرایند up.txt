دانشجو مهران کریمی درس مهندسی نرم افزار یزدان پناه
گزارش پروژه بر اساس فرایند  UP
عنوان پروژه: وب ‌اپلیکیشن تبدیل واحد
1.	مقدمه
در این پروژه یک وب‌اپلیکیشن تبدیل واحد با رابط کاربری فارسی طراحی و پیاده‌سازی شده است. هدف از انجام این پروژه، به‌کارگیری اصول مهندسی نرم‌افزار و اجرای آن بر اساس فرایند Unified Process (UP) می‌باشد. در این گزارش مراحل انجام پروژه مطابق با فازهای UP شامل Inception، Elaboration، Construction و Transition تشریح شده است.
________________________________________
2.	فاز آغازین  Inception
2-1.  هدف پروژه
هدف از این پروژه طراحی و پیاده‌سازی یک وب‌اپلیکیشن است که بتواند انواع واحدهای فیزیکی را به یکدیگر تبدیل کند و دارای رابط کاربری ساده، فارسی و کاربرپسند باشد. همچنین برخی قابلیت‌ها به صورت ویژه (Premium) در اختیار کاربران خاص قرار می‌گیرد.
2-2.  کاربران سیستم
•	کاربر عادی
•	کاربر ویژه  (Premium)
•	مدرس (ارزیاب سیستم)
2-3.  نیازمندی‌های اصلی سیستم
•	تبدیل واحدهای وزن، طول، دما، سرعت، حجم، سطح و انرژی
•	پشتیبانی از زبان فارسی و راست‌به‌چپ
•	انتخاب واحد مبدأ و مقصد
•	نمایش نتیجه تبدیل
•	سیستم ورود و خروج کاربر
•	محدودسازی تبدیل ارز برای کاربران ویژه
خروجی فاز  Inception
•	تعریف هدف پروژه
•	مشخص شدن دامنه سیستم
•	شناسایی کاربران و نیازمندی‌های کلی
________________________________________
3.	 فاز Elaboration تحلیل و طراحی
  3-1. Use Case  های سیستم
•	انتخاب نوع تبدیل
•	انتخاب واحد مبدأ
•	انتخاب واحد مقصد
•	وارد کردن مقدار
•	انجام عملیات تبدیل
•	مشاهده نتیجه
•	ورود و خروج کاربر
•	خرید اشتراک Premium
•	استفاده از تبدیل ارز
3-2.  معماری سیستم
این پروژه به صورت یک Single Page Application  و بر اساس معماری Component-Based  طراحی شده است.
فناوری‌های مورد استفاده:
•	React
•	JavaScript
•	Tailwind CSS
•	HTML
3-3.  طراحی داده‌ها (State Management)
برای مدیریت وضعیت سیستم از React Hooks استفاده شده است. مهم‌ترین stateها شامل موارد زیر هستند:
•	نوع تبدیل
•	واحد مبدأ و مقصد
•	مقدار ورودی
•	نتیجه تبدیل
•	وضعیت کاربر
•	وضعیت اشتراک Premium
3-4.  ریسک‌های پروژه
•	امکان بروز خطا در محاسبات
•	محدودیت زمان انجام پروژه
•	عدم استفاده از Backend واقعی
خروجی فاز Elaboration
•	طراحی معماری سیستم
•	مشخص شدن Use Caseها
•	شناسایی ریسک‌ها
________________________________________
4.  فاز Constructionپیاده‌سازی
4-1.  پیاده‌سازی ساختار کلی
در این فاز پروژه با استفاده از React پیاده‌سازی شده و تمام منطق سیستم در یک کامپوننت اصلی قرار گرفته است.
4-2.  پیاده‌سازی منطق تبدیل واحد
برای هر دسته از واحدها، نرخ‌های تبدیل مشخص شده و عملیات تبدیل بر اساس فرمول‌های ریاضی انجام می‌شود. تبدیل دما به صورت جداگانه و بر اساس فرمول‌های استاندارد پیاده‌سازی شده است.
4-3.  مدیریت کاربران و اشتراک Premium
سیستم ورود و خروج کاربر به صورت شبیه‌سازی‌شده پیاده‌سازی شده است. قابلیت تبدیل ارز تنها در صورت فعال بودن اشتراک Premium در دسترس کاربر قرار می‌گیرد.
4-4. طراحی رابط کاربری
رابط کاربری پروژه با استفاده از Tailwind CSS طراحی شده و دارای ویژگی‌های زیر است:
•	پشتیبانی از RTL
•	طراحی ریسپانسیو
•	سادگی و خوانایی
خروجی فاز Construction
•	پیاده‌سازی کامل سیستم
•	منطق تبدیل واحدها
•	رابط کاربری نهایی
________________________________________
5. فاز Transitionتحویل و ارائه
5-1 . تست سیستم
•	تست دستی تبدیل واحدها
•	تست ورود و خروج کاربر
•	تست قابلیت Premium

پروژه پس از انجام تست‌ها آماده تحویل نهایی و ارائه به مدرس می‌باشد.
5-3.  قابلیت توسعه
این سیستم قابلیت توسعه در آینده را دارد، از جمله:
•	اتصال به Backend واقعی
•	استفاده از API نرخ ارز
•	ذخیره اطلاعات کاربران
خروجی فاز Transition
•	پروژه آماده استفاده
•	قابل ارائه و قابل توسعه

6.  نتیجه‌گیری
در این پروژه، وب‌اپلیکیشن تبدیل واحد بر اساس فرایند UP طراحی و پیاده‌سازی شد. مراحل تحلیل، طراحی، پیاده‌سازی و تحویل به‌صورت گام‌به‌گام انجام شده و پروژه آماده ارائه نهایی می‌باشد.
